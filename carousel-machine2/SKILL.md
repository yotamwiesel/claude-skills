# /carousel-machine2 skill

Automated Instagram carousel content pipeline. Scrapes trending topics, generates Hebrew copy + slide images via Gemini (NanoBanana), and pushes to Airtable.

Source: https://github.com/peleg-jpg/carousel-machine

## Project location

`/Users/avivmalka/projects/carousel-machine2/`

## When this skill is invoked

Parse the user's command to determine the run mode, then execute the appropriate pipeline step.

## Run modes

| Command | Action |
|---|---|
| `/carousel-machine2` | Full pipeline: scrape → analyze → write → image prompts → Airtable |
| `/carousel-machine2 --research` | Research phase only (scrape + analyze + push research to Airtable) |
| `/carousel-machine2 --generate` | Generate from existing Airtable research (skip scraping) |
| `/carousel-machine2 --topic "X"` | Generate carousel for specific topic X |
| `/carousel-machine2 [YouTube URL]` | Generate carousel from a specific YouTube video |
| `/carousel-machine2 --test` | Test mode: 1 carousel only |

### Option B — Claude-native (when running as Claude Code skill)

When the user provides a YouTube URL or a topic directly, follow these steps:
1. Use the Apify scraper (`scrapers/youtube.py`) to fetch the transcript
2. Call `analysis/trend_analyzer.py` to extract the best angle
3. Call `generation/carousel_writer.py` to generate Hebrew copy
4. **STOP — Hebrew text approval step (mandatory, never skip)**
5. Call `generation/image_prompts.py` to build image prompts for Nano Banana Pro
6. Call `output/airtable.py` to push everything to Airtable

## Hebrew text approval step (mandatory)

After generating carousel copy and BEFORE generating images, present ALL Hebrew text to the user for review. Display the following in a clear, organized format:

1. **Slide count** — total number of slides
2. **Per slide** — for each slide, show:
   - Slide number and type (hook / info / cta)
   - Title (כותרת)
   - All text blocks: subtitle, body_opener, body_emphasis, body_payoff
   - Full slide text (text_he)
3. **Caption** — the full Instagram caption (caption_he)
4. **CTA line** — the caption CTA (caption_cta)
5. **Hashtags**

Then ask the user to approve, request changes, or regenerate. Only proceed to image generation after explicit user approval. If the user requests changes, apply them and present the updated text again for approval.

## Image generation model — Nano Banana Pro only

Always use **Nano Banana Pro** for generating carousel slide images. It is the best model for Hebrew text rendering and must be used exclusively — never fall back to other image generation models. All image prompts generated by `image_prompts.py` must be sent to Nano Banana Pro.

## Design system

### Cream style (default — highest performing)
- Background: `#F2EDE6` (warm cream paper texture)
- Text: `#8B2500` (dark red-brown)
- Accent: `#E8762B` (orange stickers/highlights)
- Icons: Dark charcoal rounded squares with orange line art (3D app icon style)
- Brand bar: `@aviv_malka7` on every slide
- Format: 1:1 square (1080×1080)

### Dark tech style (legacy)
- Background: `#1A1A1F` with subtle grid texture overlay (5-8% opacity)
- Primary accent: `#D4845A` (warm salmon/copper)
- Secondary: `#00E64D` (terminal green for rays/highlights)
- Format: 4:5 (1080×1350)

## Carousel structure rules

- **Slide count:** 5-10 slides (sweet spot: 7-8)
- **Slide flow:** Hook/Cover → Context/Explainer → Info slides (one concept each, 30-50 words) → Optional mid-carousel CTA → CTA slide
- Cover must be a scroll-stopper: bold text + visual, max 5 words
- Every slide must be standalone (understandable without context)
- Progress indicator on every content slide

## Hebrew copy rules (non-negotiable)

- Flowing paragraphs, NEVER bullet points on slides
- NEVER stack fragments ("יותר מהר. יותר מדויק.") = AI slop
- BANNED words: מטורף, קסם, מהפכה, פורץ דרך, שינוי משחק
- "קלוד קוד" in Hebrew when inside Hebrew text
- Plural for IG: תכפילו/שלכם (not אתה/שלך)
- CTA = invitation, not command

## Airtable schema

**Ideas & Sources table:** topic, Status, source, source_url, source_type, source_author, angle, content_pillar, engagement_score, scraped_date

**Created Images table:** slide_index (0-based), slide_type (`hook`/`info`/`CTA`), text_he, visual_prompt_used, image (attachment), status, date_created, carousel_ref
