01

The CLI Foundation

Claude Code isn't a chat window. It's an AI agent running inside your terminal with direct access to your file system, your project structure, and your code. It reads files, writes files, runs commands, and chains complex operations together - all from a single prompt.

We start here because everything else builds on it. You'll install Claude Code, understand the agent loop (how it decides what to do next), and learn the folder structure that makes it all work: the .claude/ directory, settings files, and how the tool manages permissions.

Channels (MCP Servers)

MCP (Model Context Protocol) is how Claude Code connects to external channels. Out of the box, Claude Code reads and writes local files. With MCP, it can talk to databases, APIs, documentation sites, and anything else that exposes an MCP interface.

What this means for analysts

{{DATABASE}} has its own official MCP server - Claude Code connects directly to your database. Ask in plain English, like "{{EXAMPLE_QUESTION}}" - Claude writes the query, runs it against live {{DATABASE}} data, and gives you the answer. No copy-pasting. Same for documentation: point Claude at {{DOCS_TOOL}} through MCP and it pulls context automatically.

Setting up a channel

Channels are configured in a JSON file - server type, connection details, and exposed tools. We'll connect a {{DATABASE}} MCP channel, a filesystem channel for shared reports, and a documentation channel for {{DOCS_TOOL}}. Everything respects your existing {{DATABASE}} roles and permissions - no new access is created, Claude works within the same guardrails your team already has.

05

Agents & Multi-step Workflows

When a task has more than one step, Claude Code doesn't just answer - it plans. It breaks the work into stages, checks results, and moves to the next step. This is the agent loop, and it's what separates Claude Code from a regular AI chat.

You'll learn subagents for parallel research (run three queries at once), plan mode to review Claude's approach before it executes, and Hooks for automatic actions like validating every SQL file Claude generates. We'll also cover Anthropic's model family - Opus, Sonnet, and Haiku - so you know when to use which for cost, speed, and quality.

Query optimization

Paste a slow query. Claude reads the {{DATABASE}} execution plan, identifies the bottleneck - a missing cluster key, a full table scan, an unnecessary subquery - and rewrites it. You'll see the before and after, with an explanation of what changed and why.
